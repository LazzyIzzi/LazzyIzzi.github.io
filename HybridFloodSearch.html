HybridFloodSearch and HybridFloodScan

Requires FloodFillLib.jar

Home

These plugins perform a 3D simulation of the non-wetting imbibition process in a "porosity" image by calling HybridFlood over a range of pore radii. In a porosity image, open(resolved) voxels = 1, closed(solid) voxels = 0 and everything in between 0 and 1 is unresolved porosity. See the HybridFloodFill page for more details about the HybridFlood process.

HybridFloodSearch uses a bisection search of HybridFlood results to find the critical pore radius such that the flood just reaches the back slice, sometimes referred to as "breakthrough".

Like HybridFlood, HybridFloodSearch operates only on 32-Bit stacks and supports anisotropic voxel dimensions. The search first tests if breakthrough occurs between the pore size limits. If a breakthrough threshold exists then a bisection search is done to identify the critical length.

Select a connectivity from the menu and enter a length change limit for the search. Check the options as needed. "Show Breakthrough Image" and "Show Plot" are recommended. Click "OK" to start the search.

The search continues until the largest pore radius that produces "Breakthrough" is obtained. The last row in the list is the breakthrough data. If "Show Breakthrough Image" is checked the flood data is opened in a new window.

If "Show Plot" is selected, a plot of volume vs pore radius is displayed

HybridFloodScan calls HybridFlood at fixed increments between user selected pore radii limits.

HybridFloodScan produces similar outputs the the Search plugin but it does not identify breakthrough. It is intended to provide an digital estimate of the pore(throat) size distribution similar to mercury injection capillary pressure measurements. If "Show Plot" is selected the Volume vs radius plot is displayed.

If "Save Images" was selected, the flooded images are written to tif files and can be used for visualization or other purposes.
